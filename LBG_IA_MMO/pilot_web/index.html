Astuce : tes champs, l’historique PNJ, l’état quête et l’état combat sont sauvegardés localement à chaque envoi. Chat (intention)

backend → orchestrator → agent

Astuce : si ce pilot est servi sur la VM front avec un proxy Nginx (/v1/* → backend), laisse ce champ vide. Sinon, renseigne par exemple http://192.168.0.140:8000.

Sans npc_name, l’agent utilise « PNJ ». Option world_npc_id (ex. "npc:smith") : si LBG_MMO_SERVER_URL pointe vers le serveur MMO HTTP, le backend fusionne l’état Lyra du monde avant routage. Pour enchaîner les tours, history doit être un tableau JSON valide (pas de | dans les chaînes) — ex. [{"role":"user","content":"Salut"},{"role":"assistant","content":"Bonjour."}]. Répliques générées : configurer le LLM sur le service dialogue (LBG_DIALOGUE_LLM_*, voir agents/README.md).

Exécution réelle : si LBG_DEVOPS_APPROVAL_TOKEN est défini sur l’orchestrator, ajouter devops_approval dans le JSON context (ne pas committer le secret). Clé : npc_name (sinon “global”).
